# Changelog — Kefir (MyDict)

Все заметные изменения приложения. Формат основан на
[Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование — [SemVer](https://semver.org/lang/ru/).

Версия приложения = `MARKETING_VERSION (CURRENT_PROJECT_VERSION)` из `project.yml`
и показывается в **Настройки → О приложении**. Версия встроенного контента
(набора колод-сидов) отслеживается отдельно — `SeedCoordinator.contentVersion`.

---

## [1.6] (build 7) — 2026-06-18

### Добавлено
- **«Алфавит» (полностью)** для португальского и испанского — каждая буква
  отдельной карточкой: название буквы, её звук(и) и слово-пример. Раньше была
  только горстка особых букв/сочетаний — теперь полный алфавит (PT — 26 букв,
  ES — 27, включая Ñ).
- **«Повседневные фразы»** для PT и ES — по 24 частые фразы на каждый день
  (как ты / всё в порядке / конечно / думаю да / пойдём / я устал / который час /
  ничего страшного / нет проблем …), с транскрипцией.
- Прежняя колода «Алфавит и звуки» уточнена как **«Правила чтения»** (особые
  буквы и сочетания: ç, lh, nh, ão, ñ, ll …) — для ясности.

### Изменено
- `SeedCoordinator.contentVersion` поднят до 6 — новые колоды доедут до уже
  установленного приложения при следующем запуске, без потери прогресса.

## [1.5] (build 6) — 2026-06-18

### Добавлено
- **Поддержка нескольких AI-провайдеров.** Кроме Google Gemini теперь есть
  **OpenAI (GPT)** — если один провайдер сломается или упрётся в лимит, можно
  переключиться на другой. Выбор провайдера и отдельный API-ключ — в
  **Настройки → AI-провайдер**.
- Абстракция `AIService.swift` (`AIProvider`, `AISettings`, `AIBackend`): вся
  генерация (примеры, извлечение карточек, аудит переводов, словарь) идёт через
  единый `AIBackend.generate(...)`, а он выбирает провайдера по настройке.
  Модели по умолчанию: Gemini `gemini-2.0-flash`, OpenAI `gpt-4o-mini`.

### Изменено
- `GeminiService` стал провайдер-агностичным (имя сохранено для совместимости).
- Надписи в интерфейсе «Gemini …» заменены на нейтральное «AI …».
- README переписан: добавлен раздел «Как это работает» и «Почему так».

## [1.1] (build 2) — 2026-06-18

### Исправлено
- **Встроенные колоды теперь доезжают до уже установленного приложения.**
  Раньше базовые колоды (японский, английский словарь) засеивались только при
  ПУСТОЙ базе, поэтому на обновлении новый встроенный контент сам не появлялся —
  только вручную через Настройки. Теперь синхронизация запускается на старте и
  до-добавляет недостающие встроенные колоды.
- **Прогресс больше не теряется при добавлении контента.** Синхронизация строго
  аддитивна: вставляются только колоды с новыми именами, существующие колоды,
  карточки, избранное и прогресс SRS не трогаются и не удаляются.
- Добавление/редактирование карточки вручную теперь сохраняется явно
  (`ModelContext.save()`), а не только через autosave — карточка точно не пропадёт.

### Добавлено
- `SeedCoordinator` — единая точка синхронизации встроенного контента с
  версионированием (`content_seed_version`). Чтобы новые встроенные колоды
  доехали до пользователей, достаточно поднять `SeedCoordinator.contentVersion`.
- В **Настройки → О приложении** показывается реальная версия и билд из бандла
  (вместо захардкоженной «1.0»).
- Этот файл — `CHANGELOG.md`.

### Изменено
- Кнопка **Настройки → Данные → «Загрузить встроенные колоды»** теперь делает
  полную синхронизацию (японский + английский словарь + тематические колоды и
  папки) через `SeedCoordinator.forceSync` и показывает, сколько колод добавлено.
- `Info.plist`: версия и билд берутся из `$(MARKETING_VERSION)` /
  `$(CURRENT_PROJECT_VERSION)` — единый источник правды, больше не расходятся.
