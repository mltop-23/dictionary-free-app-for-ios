## Что это

Kefir — личное iOS-приложение для зубрёжки иностранных слов. По духу — Quizlet,
но **полностью офлайн** (всё хранится на устройстве через SwiftData, без аккаунтов
и облака) и заточено под русскоязычного пользователя: переводы и **транскрипции
на русском**, чтобы запоминать звучание.

Под капотом — собственный движок интервальных повторений (**SRS, алгоритм SM-2**):
карточки сами всплывают к повтору в нужный момент, прогресс по каждому слову
отслеживается (новое → учу → повторение → освоено).

## Как это работает

**Данные.** Колоды, карточки, папки и журнал занятий — это SwiftData-модели
(`Deck`, `Card`, `Folder`, `StudyLog`), которые лежат в локальной базе на
устройстве. Один `ModelContainer` создаётся в `KefirApp` и через окружение
прокидывается во все экраны. Никакого сервера и аккаунтов нет.

**Интервальные повторения (SRS).** Каждая карточка хранит параметры SM-2:
лёгкость (`easeFactor`), интервал, число успешных повторений подряд и дату
следующего показа (`dueAt`). В режиме **Learn** ты оцениваешь, насколько легко
вспомнил слово — `SRSEngine` пересчитывает интервал: верно → интервал растёт
(дни → недели → месяцы), ошибся → карточка возвращается в начало. Стадия слова
идёт `новое → учу → повторение → освоено`. Так ты повторяешь именно то, что
вот-вот забудешь, и не тратишь время на уже выученное.

**Встроенный контент и обновления.** Готовые колоды («сиды») задаются прямо в
коде (`*SeedData.swift`, `*ThemeDecks.swift`) и словарём в `Resources/*.json`.
На старте `SeedCoordinator` сравнивает `content_seed_version` устройства со своей
`contentVersion`: если в новой сборке контента стало больше — он **до-добавляет
недостающие колоды по имени** (идемпотентно, только вставка). Твой прогресс и
собственные колоды при этом не трогаются и не удаляются.

**AI (опционально).** Генерация примеров, извлечение карточек из текста и проверка
переводов идут через единый `AIBackend.generate(prompt:)`, который выбирает
провайдера по настройке — **Google Gemini** или **OpenAI (GPT)**. Ключи хранятся
локально; без ключа приложение работает полностью, просто без AI-функций.

**Озвучка.** `SpeechService` через `AVSpeechSynthesizer` произносит карточку на
её языке (`frontLanguage` — `en-US`, `ja-JP`, `pt-BR`, `es-ES`, `ru-RU`).

## Почему так

- **Офлайн и локально** — данные не уходят в облако, приложение работает без сети
  и без аккаунта. (Бонус: не нужен платный Apple Developer для CloudKit.)
- **SRS вместо зубрёжки списком** — повторять по кривой забывания эффективнее, чем
  гонять весь список подряд.
- **Транскрипция на русском** — цель приложения «с нуля»: сначала научиться
  *произносить и узнавать* слово, а уже потом разбираться с орфографией.
- **Контент в коде + версионирование** — новые встроенные колоды доезжают до уже
  установленных копий автоматически, без миграций и без риска потерять прогресс.
- **Несколько AI-провайдеров** — если Gemini упрётся в лимит или сломается,
  переключаешься на GPT одним тапом, не трогая остальное.

## Возможности

- **5 режимов тренировки**: Карточки (flip), **Learn** (SRS с оценкой ответа),
  Match (на сопоставление), Test (тест с вариантами), Write (печатать ответ).
- **Японский режим**: отдельные поля кандзи / кана / ромадзи / значение.
- **Справочник грамматики** (📖) — таблицы по 4 языкам: времена, артикли, род,
  `ser`/`estar`, спряжения, местоимения, числа, частицы, вопросы и др.
- **Папки и подпапки** для организации колод; избранное; статистика и стадии слов.
- **Импорт из текста** и **AI-генерация** карточек/примеров — на выбор
  **Google Gemini** или **OpenAI (GPT)**; ключ хранится локально, всё опционально.
- **Озвучка (TTS)** на родном языке карточки — EN / JA / PT / ES / RU.
- **Сброс прогресса** по колоде без удаления самих карточек.

## Стек

- **Swift 5.9** + **SwiftUI** — весь UI декларативно.
- **SwiftData** — локальное персистентное хранилище (модели `Deck` / `Card` /
  `Folder` / `StudyLog`); CloudKit отключён (Personal Team).
- **AVFoundation** (`AVSpeechSynthesizer`) — озвучка карточек.
- **AI: Google Gemini / OpenAI (GPT)** — генерация примеров/карточек, аудит
  переводов (опционально, по ключу; провайдер переключается в Настройках).
- **XcodeGen** — проект (`.xcodeproj`) генерируется из `project.yml`.
- **iOS 17+**, iPhone + iPad.

## Запуск и сборка

```bash
./setup.sh                 # поставит XcodeGen (brew), сгенерит проект и откроет его
# либо вручную:
xcodegen generate
open Kefir.xcodeproj
```

Дальше в Xcode: выбрать свой Apple ID в **Signing & Capabilities**, выбрать
устройство/симулятор → **Cmd+R**.

> ⚠️ **Только полный Xcode** (не Command Line Tools). Если в терминале активны CLT,
> сборка падает с `xcodebuild requires Xcode`. Лечится:
> `sudo xcode-select -s /Applications/Xcode.app/Contents/Developer`
> (или префиксом `DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer xcodebuild …`).

> ⚠️ Версию приложения правь **только в `project.yml`** (`MARKETING_VERSION` /
> `CURRENT_PROJECT_VERSION`) — `xcodegen generate` перезаписывает `Info.plist`.

## Структура проекта

```
apps/app/
├── project.yml               # конфиг XcodeGen (версия, таргет, ресурсы)
├── setup.sh                  # xcodegen generate + open
├── CHANGELOG.md              # история версий
├── tools/                    # Python-скрипты генерации словарей/иконок
└── Kefir/
    ├── KefirApp.swift         # @main, ModelContainer, синк сидов на старте
    ├── Models/                # Deck, Card, Folder, StudyLog (SwiftData)
    ├── Views/                 # 21 экран: списки, тренировки, справочник, настройки
    ├── Services/              # движок и данные:
    │   ├── SRSEngine.swift            # алгоритм SM-2
    │   ├── SeedCoordinator.swift      # синхронизация встроенного контента
    │   ├── GrammarData.swift          # справочник (EN, JA) + .all
    │   ├── GrammarPortuguese / GrammarSpanish.swift
    │   ├── *SeedData.swift            # базовые колоды по языкам
    │   ├── *ThemeDecks.swift          # тематические колоды (PT, ES)
    │   ├── EnglishExtraDecks / EnglishLifeDecks.swift
    │   ├── GeminiService / SpeechService / ImportService …
    └── Resources/             # english_vocab.json, ipa_en.json, synonyms_en.json
```

Каждый язык самодостаточен: справочник — `extension GrammarData`, колоды — свой
`*SeedData` + `*ThemeDecks`.

## Как добавить новый язык

1. **Справочник**: новый файл `GrammarXxx.swift` — `extension GrammarData { static let xxx = LanguageSection(...) }` с топиками-таблицами; добавить `xxx` в `GrammarData.all`.
2. **Колоды**: `XxxSeedData.swift` (`seedAll(context:)`, classic-режим, `front` = слово, `back` = перевод, `example` = `[транскрипция]`, складывать в свою папку); опционально `XxxThemeDecks.swift` для тематических.
3. **Подключить**: вызвать `XxxSeedData.seedAll` в `SeedCoordinator.applyAll` (до `FolderOrganizer`) и **поднять** `SeedCoordinator.contentVersion` на +1 — чтобы контент доехал до уже установленных копий без потери прогресса.
4. `xcodegen generate` → собрать.

## Как добавить колоду

Встроенную — добавь массив в нужный `*SeedData`/`*ThemeDecks` и `insertDeck(...)`,
подняв `contentVersion`. Пользовательскую — прямо в приложении: «+» на главной,
затем «Добавить карточку» / «Импорт из текста» / «AI-генерация».
